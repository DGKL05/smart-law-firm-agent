# 课程报告素材

## 第 1 章 绪论

本项目面向律所数字化服务场景，建设一个集律所展示、律师检索、用户业务办理、后台管理与系统安全为一体的智能律所平台基础版本。当前范围聚焦基础业务系统，不接入真实 AI 对话。

## 第 2 章 相关原理与技术

- Spring Boot：用于快速构建后端 RESTful 服务。
- Vue3：用于构建动态前端页面。
- MyBatis-Plus：用于简化 MySQL CRUD、分页与逻辑删除。
- MySQL：存储核心业务数据。
- Redis：缓存热点列表和首页统计。
- MongoDB：保存操作日志，为后续 Agent 会话日志预留。
- Spring Security + JWT：实现登录认证和角色授权。

## 第 3 章 系统分析

角色分为管理员和普通用户。普通用户可浏览公开数据并管理自己的咨询、案件、预约、通知；管理员可维护系统全部业务数据。数据流从前端页面发起 Axios 请求，经 JWT 鉴权进入 Controller、Service、Mapper，再访问 MySQL；热点公开查询优先访问 Redis；操作日志写入 MongoDB。

## 第 4 章 系统总体设计

系统分为认证权限、公共展示、用户中心、管理后台、缓存日志和 Agent 预留六部分。数据库采用用户权限表、律所律师展示表、用户业务表和通知表分离设计。业务流程以登录获取 Token 为入口，用户请求携带 Token 后进入受保护接口。

## 第 5 章 系统详细设计与实现

后端按 `controller/service/mapper/entity` 分层。统一返回 `Result`，分页返回 `PageResult`。用户端 CRUD 控制器通过当前登录用户 ID 自动过滤数据；管理端接口统一位于 `/api/admin/**`。前端使用 Vue Router 区分公共页面、用户中心和管理后台，使用 Element Plus 表格、表单、弹窗和分页组件完成交互。

## 第 6 章 系统测试

测试环境为 Windows、JDK 17、Node.js、MySQL/Redis/MongoDB localhost。重点测试登录注册、权限控制、公开列表筛选、管理端 CRUD、用户端数据隔离、Redis 缓存回退和 MongoDB 日志失败不阻断业务。

## 第 7 章 总结与展望

项目完成了课程验收所需的基础业务能力、数据管理、权限控制、前后端分离和脚本材料。后续可继续接入 Dify 工作流、北大法宝 MCP 或自定义法律知识库，扩展法律咨询 Agent 和预约助手能力。
