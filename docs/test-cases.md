# 测试用例

## 登录测试

- 输入 `admin / 123456`，预期返回 JWT 和 ADMIN 角色。
- 输入错误密码，预期返回认证失败。

## 注册测试

- 输入新用户名和密码，预期创建普通用户并绑定 USER 角色。
- 重复用户名注册，预期返回“用户名已存在”。

## 律所 CRUD 测试

- ADMIN 新增、修改、删除律所，预期接口成功。
- 公开列表按省份筛选，预期只返回对应省份律所。

## 律师 CRUD 测试

- ADMIN 新增、修改、删除律师，预期接口成功。
- 公开列表按分类筛选，预期只返回对应分类律师。

## 预约 CRUD 测试

- USER 新增预约，预期记录 user_id 为当前用户。
- USER 预约已经开始的时间段，预期返回“该时间段不可预约”。
- USER 取消预约，预期状态变为“已取消”。
- 已预约记录超过咨询时间窗口后再次查询，预期状态自动变为“已完成”。
- ADMIN 分页查看预约，预期可看到所有用户预约。

## 我的咨询 CRUD 测试

- USER 新增咨询，预期状态为“待回复”。
- USER 只能查看自己的咨询。
- ADMIN 回复咨询，预期咨询状态变为“已回复”。

## 权限控制测试

- 未登录访问 `/api/me/**`，预期被拒绝。
- USER 访问 `/api/admin/**`，预期 403。
- ADMIN 访问 `/api/admin/stats`，预期成功。

## Redis 缓存测试

- 首次访问按省份律所列表，预期查询 MySQL 并写入 `lawfirm:province:*`。
- 修改律所后，预期清理相关律所缓存。
- Redis 停止时，预期公开查询仍可走 MySQL。

## MongoDB 日志测试

- 调用任意业务接口后，预期 `operation_log` 集合写入记录。
- MongoDB 停止时，预期业务接口仍正常返回，后端日志出现 warn。
